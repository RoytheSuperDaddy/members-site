# 분류 기준 참조

## 타입별 판단 기준

### events (일정)

**포함 조건 (하나라도 해당)**:
- 날짜 패턴: `YYYY-MM-DD`, `MM월 DD일`, `MM/DD`, `N월 N일`
- 키워드: 마감, 시험, 보고, 시상, 교육일, 기한, D-day, 일정, 등록일
- 문장 패턴: "~일에", "~까지", "~일 예정"

**제외 조건**:
- 날짜가 예시/설명 목적인 경우 (guides에 해당)
- 날짜 없이 절차만 설명하는 경우

---

### notices (공지)

**포함 조건 (하나라도 해당)**:
- 키워드: 안내, 변경, 정책, 발표, 공지, 시행, 개정, 업데이트
- 문장 패턴: "~을 알려드립니다", "~가 변경됩니다", "~을 안내합니다"
- 형식: 단방향 정보 전달, 수신자 행동 요구 없음

**제외 조건**:
- 단계별 절차가 포함된 경우 (guides에 해당)
- 특정 날짜 이벤트가 중심인 경우 (events에 해당)

---

### guides (가이드)

**포함 조건 (하나라도 해당)**:
- 키워드: 방법, 절차, 매뉴얼, 단계, 순서, 가이드, 스크립트, 하는 법
- 문장 패턴: "①②③", "1단계 2단계", "다음과 같이"
- 형식: 행동 유도 콘텐츠, 수신자가 따라해야 하는 절차

**제외 조건**:
- 정보 전달만인 경우 (notices에 해당)

---

## 복합 자료 처리 규칙

- 이미지 + 텍스트: 텍스트 우선, 이미지는 보조
- 날짜 + 절차 혼재: 날짜가 핵심이면 events, 절차가 핵심이면 guides
- 공지 + 절차 혼재: 절차 비중이 50% 이상이면 guides

## Confidence 산출 가이드

| 상황 | confidence 범위 |
|------|---------------|
| 키워드 3개 이상 + 패턴 명확 | 0.85 ~ 1.0 |
| 키워드 1-2개 + 문맥 지지 | 0.70 ~ 0.85 |
| 키워드 1개 or 문맥 모호 | 0.50 ~ 0.70 |
| 판단 불가 | 0.00 ~ 0.50 |
