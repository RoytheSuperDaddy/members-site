# Project Context

> CIA 에이전트가 콘텐츠 분류·추출 시 참조한다.

## 서비스 개요

- **서비스명**: N잡크루 멤버스 허브
- **URL**: `sfmi.members-center.co.kr`
- **운영자**: 삼성화재 멤버스센터 멤버스팀장 (전용)
- **사용자**: 30~60대 보험 영업 간부
- **DB**: Supabase (`vvgghtpntxffysxijkxk`)

## 콘텐츠 구조

### events (일정 테이블)

팀 운영 일정 관리. 날짜/마감/시험/교육 등 시간 기반 이벤트.

**주요 카테고리**:
- `exam`: N잡크루 자격시험, 리빌딩 시험
- `report`: 실적 보고, 현황 보고
- `reward`: 시상, 리워드 지급
- `education`: 교육, 연수
- `meeting`: 팀 회의, 본사 미팅
- `sign`: 코드등록, 서류 제출
- `general`: 기타 일정

### notices (공지 테이블)

팀원 대상 공지사항. 정책 변경, 시스템 안내, 중요 공지.

**icon_class 용도**:
- `i-blue`: 일반 공지
- `i-orange`: 리빌딩 관련
- `i-green`: 긍정적 소식 (합격, 승인 등)
- `i-purple`: 교육/학습 관련
- `i-pink`: 축하/이벤트

### guides (가이드 테이블)

업무 절차 매뉴얼. main_category로 사업부 분리.

**njob 서브카테고리**: 유입 → 학습 → 시험 → 코드등록 → 가동

**rebuild 서브카테고리**: 이관 → 친숙 → 가동 → 수수료전환 → 계속분관리

**faq**: 자주 묻는 질문 (서브카테고리 없음)

## 운영 맥락

- 콘텐츠 입력자: 멤버스팀장 1인 (운영자 = 사용자)
- 입력 빈도: 일 1~5건 예상
- 주요 입력 소재: 카톡 캡처, 사내 메일 스크린샷, 구두 메모
- 우선순위: 데이터 정확성 > 입력 속도
